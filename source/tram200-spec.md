# 札幌市電200形 輪軸・車両維持管理システム 開発仕様書

**プロジェクト名(フォルダ名)**: `tram200-wheelset-manager`


## 1. プロジェクト概要

- **目的**: 札幌市電200形の重要部検査・除籍(廃車)において、規格制限、着脱・交換履歴を追跡・検証できる維持管理実務シミュレーション環境の提供。
- **ターゲット**: 車両保守計画、維持管理業務の担当者。
- **コンセプト**: 実務第一主義(削正など余興要素は排除)。年度(R8等)と連動した履歴追跡性。

## 2. 対象車両と輪軸の互換性仕様

### 200形用 輪軸グループ
- 適合: 211, 212, 214, 221, 222, 241, 244, 246号車(計8両)
- 初期予備: 3本(`WS-200-予備1`〜`3`)
- 制約: 250形には装着不可。

### 250形用 輪軸グループ
- 適合: 251, 252, 253号車(計3両)
- 初期予備: 4本(`WS-250-予備1`〜`4`)
- 制約: 200・210・220・240形には装着不可。

## 3. 主要機能要件

- **ドラッグ&ドロップ操作**: 車両枠(4軸スロット)、予備軸保管庫、廃棄軸処分場(スクラップ)間を自由に移設。スロットに重ねると「スワップ(双方向交換)」。規格違いはエラーで弾く。
- **車両廃車プロセス**: 「廃車執行」ボタンにより即座に除籍。分離・結合いずれの状態でも常に実行可能。車両に載っていた最大4本の輪軸は自動回収され、それぞれの適正な予備保管庫に即座に転送される。
- **輪軸ステータス管理**: 輪軸のステータスは「良好」「課題あり」のみのシンプルなモデルとする(要検査/要手入れ/休止/限度到達といった段階別ステータスは設けない)。「課題あり」の内容は下記「輪軸課題管理」の課題設定によって表現する。車輪径や走行距離も管理。
- **輪軸課題管理**:
  - 各輪軸に「課題」を設定可能。複数課題の重複設定にも対応する(課題が1件もなければ「良好」扱い)。
  - 既知課題(ROM、アプリ開発時にマスタとしてテーブル化して実装):
    | 課題名 | 説明(任意) |
    |---|---|
    | 輪心溶接 | |
    | スリーブ有軸 | |
    | 乙軸 | |
  - 上記に加え、その他自由記載での課題入力も可能とする。
  - 既知課題が設定された輪軸は画面上で色分け等により一目で視認できること。
  - **色分けルール**:
    - 既知課題ごとに固定色を割り当てる(例: 輪心溶接=赤、スリーブ在軸=橙、乙軸=紫)。
    - 複数の既知課題が重複設定された場合は、輪軸アイコンを該当課題数でセグメント分割した「縞模様」表示とし、どの課題が重複しているか一目で判別できるようにする(例: 輪心溶接+乙軸 → 赤と紫の2分割表示)。
    - 自由記載課題のみが設定されている場合は、共通の注意色(例: 黄色)+アイコン(!マーク等)で表示。
    - 課題が一つも設定されていない場合は通常表示(無色/グレー)とする。
- **軸位置の呼称と移設履歴管理**:
  - 各車両の軸位置は、先頭から「車体番号-1軸」〜「車体番号-4軸」と呼称する(例: 244-3軸、211-1軸)。
  - 輪軸はID(製造番号等の個体識別子)で管理し、軸位置はあくまで「現在どこに装着されているか」を示すラベルとする。輪軸の移設に伴い、同一の輪軸が異なる軸位置呼称で呼ばれることになる。
  - 輪軸ごとに移設履歴(いつ・どの軸位置から・どの軸位置へ移設されたか)を記録し、詳細画面(台帳モーダル等)から参照できること。
  - 例: 244-3軸の輪軸を211-1軸へ移設した場合、移設後に211-1軸の詳細画面を開くと、当該輪軸が過去に244-3軸として装着されていた履歴が確認できること。
- **年度管理とリアルタイム作業日誌**: 上部で年度(R8など)を制御。操作と連動して右側に `R8 244号車廃車` のような実務テキストログをリアルタイム生成。`.txt`でエクスポート可能。
- **Undo / Redo (Ctrl+Z / Ctrl+Y)**: 誤操作を防ぐため、最大50件の操作を完全巻き戻し・やり直し可能。
- **セーブ&ロード**: 全状態を1つのJSONファイルとしてローカルに出力(保存)・入力(復元)可能。
- **車体-代車分離プロセス**:
  - 車両ごとに「車体-代車分離」ボタンを設置。押下すると分離作業の内容を設定(選択肢:「年検査」「その他(自由記載)」)。
  - 分離状態になって初めて、その車両の輪軸を予備軸と交換できる(通常は4軸とも装着済みの状態では交換不可、分離状態が交換操作の前提条件)。
  - 分離状態では同一車両内の軸間移設も可能(例: 1軸目と3軸目の輪軸を入れ替える、など)。
  - 再度「車体-代車分離」ボタンを押すと結合(分離解除)を試みる。ただし車両に装着されている輪軸が4本未満の場合は結合不可(エラーとして弾く)。例: 1軸を予備軸側に出していて3本しか装着されていない状態では結合できない。

## 4. スコープ外事項

- 「重要部検査」のタイミング判定(走行距離・経過年数での自動アラート)は対象外。あくまで輪軸の付け替えがシミュレートできれば良く、検査タイミングの自動判定機能は持たない。

## 5. 運用ルール・制約事項

- **初期データ**: アプリ起動時の各輪軸の状態(課題・予備軸の在庫状況等)は、特定の初期データを投入する。具体的な初期データ内容は別途指定する(下記6章「初期データ詳細」参照、または別ファイルで提供)。
- **同時分離数の制限**: 「車体-代車分離」状態にできる車両は、同時に1両のみとする(現実の作業ヤード制約を反映)。既に分離中の車両がある状態で別車両の分離を試みた場合はエラーとする。
- **スクラップ処分の扱い**: 一度「廃棄(スクラップ)」状態になった輪軸は復活させない(廃棄済みとして固定)。ただし、誤操作時はUndo機能による直前操作の取り消しは可能とする。
- **規格違い操作時のエラー**: 規格不適合な移設(例: 200形用輪軸を250形車両へ)を試みた場合、操作を無効化(元の位置に戻す)した上で、エラーメッセージを表示する。
- **実装形態**: 単一HTMLファイル(JS/CSS含む、ローカルで開いて動作)として実装する。
